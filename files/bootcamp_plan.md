# HACKER PROTOCOL BOOTCAMP (HPB)

## Internal Plan — Minimal Requirements

---

## Objective

Deliver a beginner-focused cybersecurity bootcamp using a minimal, controlled toolset. The focus is on system understanding, observation, and practical thinking rather than tool dependency.

---

## Core Principle

The bootcamp operates on a minimal stack.

Priority:

* Understand systems
* Observe behavior
* Practice consistently

No tool overload.

---

## Required Setup (Pre-Week 1)

All students must have the following:

### 1. Web Browser

* Google Chrome (or any modern browser)
* Used for inspection and network analysis via DevTools

### 2. Terminal

* Default system terminal (Linux/Mac)
* Git Bash (Windows)

### 3. Code Editor

* VS Code
* Used for notes and basic file interaction

### 4. Notes

* Any note-taking method (Markdown, Notion, Obsidian, or plain text)
* Must be used consistently

## Exclusions

The following are not required:

* Advanced hacking frameworks
* Paid tools
* Complex scripts
* Multiple operating systems

---

## Learning Rules

* Understand before using tools
* Practice consistently
* Maintain structured notes
* Ask relevant questions
* Operate within ethical boundaries

---

## Ethics

This program is strictly for:

* Learning
* Practice
* Skill development

Not permitted:

* Attacks on real systems
* Illegal activity

---

## Final Position

The goal is not tool mastery.

The goal is:

* System understanding
* Critical thinking
* Problem solving

Tools are secondary.
